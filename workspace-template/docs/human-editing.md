# Human Editing and Explicit Snapshots

Research workspaces are designed for both human editing and agent automation.
Human edits must be visible before broad agent operations, but snapshots must be explicit. Git hooks must not create nested commits or silently add broad paths.

## Editable Areas

Humans may normally edit:

- `wiki/` maintained research pages;
- `index.md` and `log.md`;
- `docs/` operating documentation;
- `skills/` agent playbooks.

Raw evidence and generated source layers are not part of the default human-edit snapshot scope:

- `raw/` stays immutable;
- `sources/manifest.jsonl` is generated by inventory tooling;
- `sources/normalized/` is generated by normalization tooling;
- `sources/cards/` is generated or derived source metadata.

## Snapshot Dry Run

Before broad agent operations, run the snapshot script from the workspace root:

```bash
python3 scripts/snapshot_user_edits.py
```

The dry run prints scoped changed paths and the commit command that would be used. It does not stage files and does not commit.

Use `--path` to include an additional human-editable path:

```bash
python3 scripts/snapshot_user_edits.py --path AGENTS.md
```

## Explicit Snapshot Commit

Create a snapshot only when the user explicitly asks for it:

```bash
python3 scripts/snapshot_user_edits.py --commit --message "snapshot: user edits before ingestion"
```

The script stages only scoped paths and commits them with the supplied message. If no scoped user edits exist, it creates no commit.

The script refuses to commit when the git index already has staged changes.
Commit or unstage existing index changes before snapshotting user edits, so a snapshot cannot accidentally include unrelated work.

## Hook Policy

Do not install or rely on git hooks that run `git commit`, auto-add broad paths, or create nested commits. Hooks may be useful for local checks, but user-edit snapshots are explicit commands, not implicit side effects.

To check for active non-sample hooks:

```bash
find .git/hooks -maxdepth 1 -type f ! -name '*.sample' -print
```

An empty result means no active repository hook files are installed.

## Agent Workflow

Agents should:

1. run `python3 scripts/snapshot_user_edits.py` before broad operations;
2. show the user any scoped edits that would be snapshotted;
3. use `--commit` only when explicitly requested;
4. proceed without a snapshot when the dry run reports no scoped user edits.
